# X投稿作成スキル

記事の内容からXに投稿する文章を作成し、`bin/post-to-x.php` で投稿する。
投稿記録は `data/x-posts.json` で管理される。

## 投稿戦略（Xアルゴリズム最適化済み）

### 構成（2ステップ投稿）
1. **1投稿目**：サムネイル画像 + ヒキのある文章（外部リンクなし）
2. **リプ**：記事リンク（1投稿目の直後に自動投稿）

外部リンクを本文に入れるとXアルゴリズムでリーチがほぼゼロになるため、リンクは必ずリプに分離する。

### 1投稿目の書き方ルール

**フック（冒頭）：**
- 記事タイトルの【速報】【朗報】【悲報】【炎上】タグはそのまま活かす
- 数字・固有名詞・感情を冒頭に持ってくる
- 「これマジか」「夢ある」「終わった」など2ch/なんJ的な感嘆を入れる

**本文：**
- 記事の核心情報を1〜2行で要約する（全角80〜120文字が目安）
- 改行を使って視認性を上げる
- 読者が「リプ見よう」「RT/引用したくなる」内容にする

**ハッシュタグ：**
- 必ず2個（カテゴリ系1個 + `#オリパ`）
- カテゴリ系: `#ポケカ` / `#遊戯王` / `#ワンピース` / `#トレカ`

**禁止事項：**
- 1投稿目にURLを入れない
- ハッシュタグ3個以上
- 絵文字3個以上
- 同一内容のコピペ投稿（シャドウバンリスク）

### リプの書き方
```
▼ 詳細はこちら
{記事URL}
```

### 投稿テンプレート例

```
【速報】cardéria池袋店がPSA10確定ポケカオリパを発売

ニンフィアVMAX SAやミュウVMAX SAが確定で入ってるとか夢ありすぎる…
33,000円と25,000円の2種、各50口限定

#ポケカ #オリパ
```

## 実行手順

### 1. 未投稿記事を確認する

```bash
php bin/post-to-x.php --list        # 未投稿記事一覧
php bin/post-to-x.php --list=all    # 全記事の投稿状態（投稿済み含む）
```

### 2. 投稿対象の記事を選ぶ

- ユーザーが指定した場合はそのslugを使う
- 指定がなければ `--list` の未投稿一覧から提案する
- 既に投稿済み（`data/x-posts.json` に記録あり）の記事は重複注意を伝える

### 3. 記事を読んで投稿文を作成する

`content/articles/{slug}.md` を読み、上記の書き方ルールに従って投稿文を作成する。
記事タイトルをそのまま使うのではなく、核心情報を抽出してヒキのある文章にリライトすること。

### 4. dry-run でプレビュー確認

```bash
php bin/post-to-x.php --slug={slug} --text="投稿文" --reply="リプ文" --dry-run
```

**必ずユーザーの確認を取ってから本番投稿に進むこと。**

### 5. 本番投稿

```bash
php bin/post-to-x.php --slug={slug} --text="投稿文" --reply="リプ文"
```

投稿が成功すると、自動的に `data/x-posts.json` に以下が記録される：
- `slug` — 記事slug
- `tweet_id` — ツイートID
- `reply_id` — リプライID
- `text` — 1投稿目の本文
- `reply_text` — リプの本文
- `posted_at` — 投稿日時（JST ISO 8601）

### 6. 投稿結果のURLをユーザーに共有する

投稿完了後、ツイートURLを表示する：
```
https://x.com/i/status/{tweet_id}
```

## 投稿管理JSON（data/x-posts.json）

### 構造
```json
{
  "posts": [
    {
      "slug": "carderia-psa10-pokeka-oripa",
      "tweet_id": "2040374404170014853",
      "reply_id": "2040374405629706333",
      "text": "投稿本文...",
      "reply_text": "▼ 詳細はこちら\nhttps://...",
      "posted_at": "2026-04-04T12:00:00+09:00"
    }
  ]
}
```

### 用途
- 投稿済み/未投稿の管理（重複投稿防止）
- 過去の投稿文の参照（文体の一貫性維持）
- 投稿パフォーマンス分析の基礎データ

## 注意

- 投稿前に必ず `--dry-run` でプレビューしてユーザー確認を取ること
- 記事のサムネイル画像（`thumbnail_url`）がある場合は自動で画像添付される
- `config/twitter.php` が必要（APIキー設定ファイル、gitignore済み）
- `bin/review-drafts.php --publish` でも自動投稿＋JSON記録が行われる
